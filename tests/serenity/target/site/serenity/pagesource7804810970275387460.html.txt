<h1 id="getting-started">GETTING STARTED</h1>

<p class="lead double-margin">Everything you need to get up and running with the Ticketmaster API. Basic concepts are talked about here, so please keep reading :)</p>

<h2 id="introduction">Introduction</h2>

<p class="body">To get the most out of your experience, <a href="https://developer-acct.ticketmaster.com/user/register">register for an API</a> or <a href="https://developer-acct.ticketmaster.com/user/login">log in to your account</a> now. We’ll render links in examples and code samples into active link using your own API Key. If you prefer to jump right into the APIs and make live calls, check out the <a href="/api-explorer/">API Explorer</a>.</p>

<p>We currently offer event discovery and commerce APIs with various <a href="/products-and-docs/apis/getting-started/#available-resources">access tiers</a>. Upon registration and obtaining your API key, you will be able to access our <a href="/products-and-docs/apis/discovery-api/v2/">Discovery</a> and <a href="/products-and-docs/apis/commerce/">Commerce</a> APIs instantly. Using both APIs allows you to create a meaningful event discovery experience for your fans.</p>

<p><em>Note: The <a href="/products-and-docs/apis/international-discovery/">International Discovery API</a> is currently being consolidated with the <a href="/products-and-docs/apis/discovery-api/v2/">Discovery API</a>, therefore we recommend that developers plan to use this single service for access to global events.</em></p>

<p>Our APIs work against many platforms including Ticketmaster, TicketWeb, Universe, FrontGate, Ticketmaster Resale (TMR) and many more.</p>

<p>Event coverage is global.</p>

<h2 id="rate-limit">Rate Limit</h2>

<p>All API keys are issued with a <em>default quota</em> of <strong>5000 API calls/day</strong>. We do increase rate limits on case-by-case basis. In order to increase the rate limit for a particular application, we need to verify the following:</p>

<ol>
  <li>The application is in compliance with our <a href="/support/terms-of-use/">Terms of Service</a></li>
  <li>The application is in compliance with our branding guide</li>
  <li>The application is representing the Ticketmaster data properly</li>
</ol>

<p>Once these three criteria are verified, the rate limit is increased to what Ticketmaster and the developer determine to be appropriate.</p>

<h3 id="rate-limit-info-in-response-header">Rate Limit Info in Response Header</h3>
<p>You can see how much of your quota has been used by checking the following <strong>response headers</strong>:</p>

<ul>
  <li><strong>Rate-Limit</strong>: What’s the rate limit available to you. The default is 5000.</li>
  <li><strong>Rate-Limit-Available</strong>: How many requests are available to you. This will be 5000 minus all the requests you’ve done.</li>
  <li><strong>Rate-Limit-Over</strong>: How many requests over your quota you’ve made.</li>
  <li><strong>Rate-Limit-Reset</strong>: The UTC date and time of when your quota will be reset.</li>
</ul>

<figure class="highlight"><pre><code class="language-bash" data-lang="bash">curl -I <span class="s1">'http://app.ticketmaster.com/discovery/v1/events.json?keyword=Queen&amp;apikey=xxx'</span>

HTTP/1.1 200 OK
Rate-Limit: 5000
Rate-Limit-Available: 4978
Rate-Limit-Over: 0
Rate-Limit-Reset: 1453180594367</code></pre></figure>

<h2 id="cors-support">CORS Support</h2>

<p>The API also supports <a href="http://en.wikipedia.org/wiki/Cross-origin_resource_sharing" target="_blank">Cross-Origin Resource Sharing (CORS)</a> which allows cross-domain requests to be made by JavaScript on a web page. Such “cross-domain” requests would otherwise be forbidden by web browsers, per the <a href="http://en.wikipedia.org/wiki/Same_origin_policy" target="_blank">same origin security policy</a>. CORS is supported by all modern web browsers, and a full list of browser support can be found <a href="http://caniuse.com/cors" target="_blank">here</a>.</p>
